# Contributing

Thank you for your interest in contributing to Docscribe!

## How to contribute

### Reporting bugs

Open an issue with a minimal reproduction. Include:
- Ruby version
- Gem versions (docscribe, parser, rbs/sorbet if relevant)
- Input code
- Expected vs actual output

### Suggesting features

Open an issue describing the use case and desired behaviour.

### Pull requests

1. Fork and create a branch from `master`.
2. Run `bundle install` and ensure tests pass: `bundle exec rspec`.
3. Run `bundle exec rubocop` — zero offenses required.
4. If you change type providers, run `bundle exec steep check` (Ruby ≥ 3.2).
5. Run `bundle exec docscribe lib -C docscribe.yml` to verify no doc drift.
6. Open a PR targeting `master`.

## Development setup

```bash
git clone <your-fork>
cd docscribe
bundle install
bundle exec rspec
```

For RBS/steep work (requires Ruby ≥ 3.2):
```bash
bundle install --with rbs
bundle exec steep check
```

## Code conventions

- Target Ruby 2.7+.
- Follow existing code style (RuboCop enforces it).
- YARD-style `@param` / `@return` documentation is autogenerated — do not edit inline doc blocks manually.
- For type hints, prefer `#:` (steep) annotations where safe as trailing comments.
